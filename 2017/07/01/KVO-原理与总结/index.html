KVO 原理与总结

KVO key value observe 观察者模式
观察者 被观察的对象属性 移除观察者

iOS系统用的KVO的地方

NSOperation
NSOperationQueue

第三方用到KVO的地方

RAC 用的特别多
MJRefresh 监听contentoffset 
所有的第三方视频播放器都会用KVO检测进度

一般都是用自动开启，也可以手动开启KVO

以上是实验的结果，可以看到在手动模式即使不给属性赋值，也能触发KVO，所以可以得出结论：KVO不是监听地址变化，也不是监听值得变化，关键是调用了willChangeValueForKey、didChangeValueForKey，还有automaticallyNotifiesObserversForKey的返回值。

修改以上测试代码

自动模式下类名发生了变化，所以KVO得原理就是：利用运行时，生成一个对象的子类，并生成子类的对象，替换原来对象的isa指针，并重写了set方法，在set方法里添加了willChangeValueForKey、didChangeValueForKey这两个方法。KVO是通过kvc去找方法和属性的
